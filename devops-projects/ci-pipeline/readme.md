# CI/CD Pipeline for C++ Project with Google Test

Этот проект демонстрирует настройку **непрерывной интеграции (CI)** для C++ приложения с использованием **GitLab CI**, **CMake** и **Google Test**.  
Пайплайн автоматически собирает проект, запускает unit-тесты и формирует артефакты для развёртывания.

---

## Технологический стек

| Технология      | Назначение                              |
|-----------------|-----------------------------------------|
| C++17           | Язык программирования                   |
| CMake 3.5+      | Система сборки                          |
| Google Test     | Фреймворк для модульного тестирования   |
| GitLab CI       | CI/CD платформа                         |
| Ubuntu 24.04    | Среда выполнения (Docker‑образ)         |

## Описание компонентов

### `B` – библиотека `ClassLib`

Содержит общую логику, которая используется основной программой и тестами.  
Собирается как динамическая библиотека (`libClassLib.so`).

### `C` – исполняемый модуль и тесты

**`C`** – основная программа, использующая `ClassLib`.

---

## CI/CD пайплайн (GitLab CI)

Пайплайн состоит из трёх стадий, выполняемых последовательно:

| Стадия      | Задача                                                                 |
|-------------|------------------------------------------------------------------------|
| **build**   | Сборка проекта с помощью CMake, установка артефактов в `exe/bin` и `exe/lib`, сохранение тестового бинарника.                                                                   |
| **test**    | Запуск `CTestExe` с генерацией JUnit-отчёта, загрузка отчёта в GitLab для отображения результатов тестов.                                                                    |
| **deliver** | Копирование собранных файлов в `dist/exe` и сохранение их как финального артефакта (доступен для скачивания в течение недели).                                            |

**Ключевые особенности:**

- Google Test клонируется автоматически в папку `3rdparty` на этапе `before_script`.
- Для поиска динамической библиотеки используется `LD_LIBRARY_PATH`.
- Результаты тестов отображаются в интерфейсе GitLab (вкладка **Tests**).
