# Tuple

Реализован шаблонный класс с переменным количеством аргументов - `Tuple` (кортеж), обобщение класса `std::pair`, аналог `std::tuple` из С++11.
Класс обладает следующей функциональностью:

- Конструктор по умолчанию (все элементы кортежа инициализируются значениями по умолчанию). Этот конструктор участвует в перегрузке тогда и только тогда, когда `std::is_default_constructible<Ti>::value` верно для всех `i`. Этот конструктор explicit в том и только том случае, когда `Ti` не является `copy-list-initializable` из $\{\}$ для какого-либо `i`.
- Конструктор из набора аргументов соответствующих типов для Tuple (конструктор $(2)$ со страницы [tuple](https://en.cppreference.com/w/cpp/utility/tuple/tuple)). Этот конструктор участвует в перегрузке, только если `std::is_copy_constructible<Ti>::value` верно для всех `i`. Этот конструктор `explicit`, если `std::is_convertible<const Ti&, Ti>::value` неверно хотя бы для одного `i`.
- Конструктор от универсального набора аргументов `UTypes&&... args` (конструктор $(3)$ со страницы [tuple](https://en.cppreference.com/w/cpp/utility/tuple/tuple)). Этот конструктор участвует в перегрузке, только если размеры пакетов аргументов совпадают, эти размеры больше единицы, и `std::is_constructible<Ti, Ui>::value` верно для всех `i`. Этот конструктор `explicit`, если `std::is_convertible<Ti, Ui>::value` неверно хотя бы для одного `i`.
- Конструкторы от другого `Tuple<UTypes> other`, причем поддержаны версия $(5)$ от `const Tuple<UTypes>&` и версия $(6)$ от `Tuple<UTypes>&&`. Эти конструкторы инициализируют каждый элемент `Tuple` выражением `get<i>(FWD(other))`, где `FWD(other)` означает `std::forward<decltype(other)>(other)`.

Эти конструкторы участвуют в перегрузке, только если выполнены все условия:

- Размеры пакетов `Types` и `UTypes` совпадают;
- `std::is_constructible_v<Ti, decltype(get<i>(FWD(other)))>` верно для всех `i`;
- Или размер пакета `Types` не равен `1`, или `std::is_convertible_v<decltype(other), T>, std::is_constructible_v<T, decltype(other)>` и `std::is_same_v<T, U>` все имеют значение `false`.

Эти конструкторы `explicit`, если `std::is_convertible_v<decltype(std::get<i>(FWD(other))), Ti>` неверно хотя бы для одного `i`.

- Конструкторы от `std::pair<T1, T2>`, которые создают `Tuple<T1, T2>`. Поддержана версия от `const std::pair<T1, T2>&` и от `std::pair<T1, T2>&&`. Для корректной работы этих конструкторов написаны template deduction guides.
- Конструкторы копирования и перемещения `Tuple`. Конструктор копирования кидает CE, если `std::is_copy_constructible<Ti>::value` неверно для какого-либо `i`. Конструктор перемещения не участвует в перегрузке, если `std::is_move_constructible<Ti>::value` неверно для какого-либо `i`.
- Оператор копирующего присваивания `Tuple`. Если `std::is_copy_assignable<Ti>::value` неверно хотя бы для одного `i`, то его вызов приводит к CE.
- Оператор перемещающего присваивания. Присваивает каждому элементу `Tuple` выражение `std::forward<Ti>(get<i>(other))`. участвует в перегрузке, только если `std::is_move_assignable<Ti>::value` верно для всех `i`.
- Оператор присваивания от `const tuple<UTypes...>& other`. участвует в перегрузке, только если размеры пакетов совпадают и `std::is_assignable<Ti&, const Ui&>::value` верно для всех `i`.
- Оператор присваивания от `Tuple<UTypes...>&& other`. участвует в перегрузке, только если размеры пакетов совпадают и `std::is_assignable<Ti&, Ui>::value` верно для всех `i`.
- Операторы присваивания от пары `const std::pair<T1, T2>&` и от `std::pair<T1, T2>&&`.

Помимо этого, реализованы функции, не являющиеся членами класса:

- Функции `MakeTuple`, `Tie` и `ForwardAsTuple`, создающие новый `Tuple` аналогично тому, как это делают функции `std::make_tuple`, `std::tie` и `std::forward_as_tuple`.
- Функция `Get` с шаблонным параметром `size_t Index`, которая принимает `Tuple` и возвращает ссылку на `Index`-й элемент кортежа. Причем ссылка того же вида, что была и ссылка на принятый `Tuple`.
- Функция `Get` с шаблонным параметром `T`, которая принимает `Tuple` и возвращает ссылку на тот элемент кортежа, который имеет тип `T`. Причем ссылка того же вида, что была и ссылка на принятый `Tuple` (lvalue, const lvalue или rvalue). Если в `Tuple` нет элемента типа `T` или таких элементов несколько, то вызов `Get<T>` приводит к ошибке компиляции.
- Функция `TupleCat`, которая возвращает кортеж, являющийся конкатенацией нескольких кортежей, переданных в качестве аргументов. При этом, если исходный кортеж был lvalue, то его элементы будут скопированы, а если rvalue - то мувнуты в новоиспеченный кортеж. За время работы `TupleCat` элементы исходных кортежей копируются или муваются не более одного раза.
- Лексикографическое сравнение `Tuple`.

### Запуск тестов

осуществляется следующими командами:

```bash
cd cpp-projects && mkdir build
cmake -S . -B build
cmake --build build/ --target test_tuple && build/test_tuple
```
